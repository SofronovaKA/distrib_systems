# Лабораторная работа 3.1. Организация асинхронного взаимодействия микросервисов с помощью брокера сообщений

## Постановка задачи

Разработать распределенную систему управления умным домом, способную обрабатывать три типа задач:
1. Управление устройствами умного дома (включение/выключение света)
2. Конвертация текста из формата Markdown в HTML
3. Удаление дубликатов из списка слов

Система должна поддерживать как синхронное взаимодействие через gRPC, так и асинхронное через брокер сообщений RabbitMQ, обеспечивая надежную доставку и обработку запросов.

## Вариант 17

**Три задания:**
1. **Управление умным домом.** Producer отправляет команду "включить свет". gRPC сервис обрабатывает ее и возвращает "Свет на кухне включен".
2. **Конвертер Markdown в HTML.** Producer отправляет текст в Markdown. gRPC сервис конвертирует его в HTML (библиотека markdown) и возвращает результат.
3. **Удаление дубликатов.** Producer отправляет список слов. gRPC сервис удаляет дубликаты и возвращает уникальные слова.

## Архитектура

### Часть 1. Синхронное взаимодействие (gRPC)

Синхронное взаимодействие реализовано по модели клиент-сервер с использованием gRPC. Клиент отправляет запрос и блокируется до получения ответа от сервера.

<img width="666" height="356" alt="image" src="https://github.com/user-attachments/assets/3a17e4de-4156-4c76-ab7f-265bf2de6526" />

**Компоненты:**
- **gRPC клиент (`grpc_client.py`)** — отправляет команды серверу в интерактивном режиме, ожидает синхронного ответа
- **gRPC сервер (`grpc_server.py`)** — обрабатывает запросы и возвращает результаты

**Описание взаимодействия**

1. **Установка соединения** — Клиент устанавливает gRPC канал с сервером по адресу `localhost:50051`
2. **Отправка запроса (Вопрос)** — Клиент вызывает удаленный метод `ControlDevice`, передавая команду и местоположение
3. **Обработка запроса** — Сервер получает запрос, выполняет бизнес-логику (включение/выключение света)
4. **Возврат ответа** — Сервер формирует ответ с результатом операции и отправляет клиенту
5. **Завершение** — Клиент получает ответ, выводит результат и закрывает соединение

**Пример взаимодействия**

```bash
# Клиент отправляет запрос
Введите команду: включить свет
Введите место: кухня

# Сервер обрабатывает
[СЕРВЕР] Получен запрос: включить свет в кухня
[СЕРВЕР] Ответ: Свет на кухне включен

# Клиент получает ответ
Ответ сервера: Свет на кухне включен
Статус: УСПЕШНО ✓
```

### Часть 2. Асинхронное взаимодействие (RabbitMQ)

Асинхронное взаимодействие реализовано через брокер сообщений RabbitMQ, обеспечивающий надежную очередь задач.

<img width="674" height="285" alt="image" src="https://github.com/user-attachments/assets/4b079d48-236e-46b3-9caf-9a9b0090a9dc" /> 

**Компоненты:**

| Компонент | Файл | Функция |
|-----------|------|---------|
| **Producer** | `producer.py` | Отправляет задания в очередь RabbitMQ |
| **RabbitMQ** | Docker контейнер | Хранит сообщения в очереди, обеспечивает надежную доставку |
| **Consumer** | `consumer.py` | Забирает сообщения из очереди, вызывает gRPC сервер |
| **gRPC Server** | `grpc_server.py` | Обрабатывает команды умного дома |

**Преимущества асинхронного подхода**

| Характеристика | Описание |
|----------------|----------|
| **Разделение компонентов** | Producer и Consumer не зависят друг от друга, могут работать в разное время |
| **Устойчивость к нагрузкам** | При пиковых нагрузках задачи накапливаются в очереди, не теряясь |
| **Надежность** | Персистентное хранение гарантирует сохранность сообщений |
| **Масштабируемость** | Можно добавить несколько Consumer'ов для параллельной обработки |
| **Визуализация** | Веб-интерфейс RabbitMQ позволяет отслеживать состояние очереди |

### Файл docker-compose.yml

```yaml
version: '3.8'

services:
  rabbitmq:
    image: rabbitmq:3.9-management
    container_name: rabbitmq
    ports:
      - "5672:5672"      # Порт для подключения клиентов
      - "15672:15672"    # Порт для веб-интерфейса
    environment:
      - RABBITMQ_DEFAULT_USER=user
      - RABBITMQ_DEFAULT_PASS=password
```

## Запуск системы

### 1. Запуск RabbitMQ (Docker)

```bash
docker run -d 9d02640acc7d rabbitmq -p 5672:5672 -p 15672:15672 \
  -e RABBITMQ_DEFAULT_USER=user -e RABBITMQ_DEFAULT_PASS=password \
  rabbitmq:3.9-management
```

```bash
docker-compose up -d
```

### 2. Запуск gRPC сервера

```bash
cd grpc_sync
python3 grpc_server.py
```

### 3. Запуск Consumer

```bash
cd rabbitmq_async
python3 consumer.py
```

### 4. Запуск Producer (отправка заданий)

```bash
cd rabbitmq_async
python3 producer.py
```

### 5. Мониторинг через веб-интерфейс

```text
http://localhost:15672
Логин: user
Пароль: password
```

- Асинхронный (RabbitMQ) — обеспечивает надежность, масштабируемость и разделение компонентов.
